# Rico MD vNext PRD-2

> 版本：vNext / PRD-2  
> 更新日期：2026-04-17  
> 对应方向：文档管理补完 + 写作效率提升 + 代码块体验修正

---

## 1. 背景

当前项目已经完成模块化重构、图片本地存储、主题系统、基础多文档切换、公众号复制链路等核心能力，但仍有三个明显短板：

1. 多文档仅达到“可切换”，还不具备完整管理能力。
2. 编辑区仍然偏底层，常见 Markdown 操作需要用户手写。
3. 代码块预览、主题切换和公众号导出之间存在不一致。

本 PRD-2 的目标是把编辑器从“可用工具”推进为“可长期写作的工作台”。

---

## 2. 目标

### 2.1 产品目标

- 把多文档功能补齐到可管理、可搜索、可维护。
- 把高频 Markdown 操作前置到可点击的工具条。
- 让代码块在预览和导出到公众号时保持一致，并支持可配置展示。

### 2.2 成功标准

- 用户可在左侧完成文档搜索、重命名、复制、删除、上移、下移。
- 用户可为当前文档显式设置标题，不再完全依赖正文 `# 一级标题`。
- 编辑区支持点击插入标题、引用、代码块、分割线、图片、表格等常用 Markdown 结构。
- 底部状态栏明确展示 `保存中 / 已保存 / 保存失败`。
- 代码块主题切换后，预览和导出后的公众号 HTML 视觉一致。

## 4. 主要交互

### 4.1 文档管理

- 左侧边栏新增搜索框。
- 文档列表默认按 `updatedAt` 倒序展示。
- 每个文档项显示：
  - 标题
  - 最近保存时间
  - 未保存状态点
  - 操作按钮：重命名 / 复制 / 上移 / 下移 / 删除
- 删除当前文档前必须二次确认。
- 删除后自动切换到最近的可用文档。

### 4.2 当前文档标题

- 编辑区顶部提供独立标题输入框。
- 标题输入框更新的是文档显式标题 `manualTitle`。
- 显式标题不自动回写正文中的 `# 一级标题`。
- 文档显示标题解析规则：
  1. `manualTitle`
  2. 正文第一个 `# 标题`
  3. `未命名文档 N`

### 4.3 写作工具条

- 工具条位于标题输入框下方、编辑区上方。
- 提供如下快捷插入：
  - H1
  - H2
  - H3
  - 加粗
  - 斜体
  - 引用
  - 代码块
  - 分割线
  - 图片语法
  - 上传图片
  - 表格
- 插入逻辑要求：
  - 有选区时优先包裹或替换选区
  - 无选区时按模板插入
  - 插入后光标跳到最常编辑的位置

### 4.4 保存状态

- 输入后立即进入 `saving`
- 持久化成功进入 `saved`
- localStorage 写入失败进入 `error`
- 底部状态栏显示：
  - 保存状态
  - 字符数 / 词数 / 阅读时长
  - 最后保存时间

### 4.5 代码块体验

- 代码块支持结构化头部：
  - 语言标签
  - 复制按钮
  - macOS 三色装饰
- 三项能力均为全局设置，不按文档保存。
- 若三项全部关闭，则代码块头部整体隐藏。
- 复制按钮仅在预览区生效，不进入导出结果。

## 5. 数据结构

### 5.1 文档对象

```js
{
  id: string,
  manualTitle: string,
  title: string,
  content: string,
  createdAt: number,
  updatedAt: number,
  sortOrder: number,
  dirty: boolean
}
```

说明：

- `title` 保留为兼容字段。
- 新逻辑优先使用 `manualTitle` 作为文档显式标题。
- `sortOrder` 用于上移/下移和未来拖拽排序的接口预留。
- `dirty` 只表示该文档存在未持久化变更。

### 5.2 全局代码块设置

```js
{
  showLanguageLabel: boolean,
  showCopyButton: boolean,
  showMacDecorations: boolean
}
```

### 5.3 当前活动文档状态

运行态不写入单文档对象，放在主状态里：

```js
{
  currentSaveState: 'saving' | 'saved' | 'error',
  lastSavedAt: number | null
}
```

## 6. 持久化策略

### 6.1 localStorage key

- `currentStyle`
- `markdownInput`
- `documents`
- `activeDocumentId`
- `codeBlockSettings`

### 6.2 兼容策略

- 若历史 `documents` 项缺少 `manualTitle / sortOrder / dirty`，加载时自动补默认值。
- 若没有 `documents`，但存在旧版 `markdownInput`，则迁移为一个默认文档。
- 继续保留旧版 `markdownInput` 写入，以保持向后兼容。

## 7. 技术实现要求

### 7.1 `preferences.js`

- 负责：
  - 读取和规范化文档数组
  - 读取和规范化 `codeBlockSettings`
  - 保存当前主题、当前文档内容、文档数组、活动文档 ID、代码块设置
  - 防抖保存成功/失败回调

### 7.2 `main.js`

- 负责：
  - 文档列表筛选、排序、切换
  - 文档操作：新增、重命名、复制、删除、上移、下移
  - 显式标题输入
  - 自动保存状态机
  - 工具条插入逻辑
  - 预览点击事件中的“复制代码”
  - 代码块设置与主题切换的联动

### 7.3 `markdown-engine.js`

- 输出结构化代码块 DOM：
  - `.md-code-block`
  - `.md-code-block-header`
  - `.md-code-block-language`
  - `.md-code-block-copy`
  - `.md-code-block-body`
  - `.md-code-block-code`
- 不在这里写死最终主题色。

### 7.4 `render-pipeline.js`

- 接收：
  - `codeTheme`
  - `codeBlockSettings`
- 对代码块结构应用最终视觉样式。
- 负责根据全局设置显示/隐藏语言标签、复制按钮、macOS 装饰。

### 7.5 `clipboard-exporter.js`

- 导出时读取代码块结构，而不是回退到固定 One Dark。
- 保留当前代码主题的：
  - 背景色
  - 边框
  - 文字颜色
- 导出时删除复制按钮。
- 导出时语言标签和 macOS 装饰是否保留，跟全局设置一致。

## 8. 验收标准

### 8.1 文档管理

- 可以新增多个文档
- 可以搜索显式标题和正文内容
- 可以复制当前文档
- 可以删除文档并自动切换
- 可以上移/下移文档
- 刷新后元数据恢复正确

### 8.2 保存状态

- 输入后立刻显示“保存中”
- 防抖成功后显示“已保存”
- 写入失败时显示“保存失败”
- 最后保存时间基于真正持久化成功时间

### 8.3 写作工具条

- 每个插入按钮都能正确写入 Markdown
- 光标位置正确
- 图片上传按钮与现有图片压缩/存储链兼容

### 8.4 代码块

- 切换 6 个代码主题时，预览立即变化
- 语言标签/复制按钮/macOS 装饰开关立即生效
- 点击复制按钮可复制纯代码
- 导出到公众号后，代码块视觉与预览一致

### 8.5 回归

- 主题切换不受影响
- 图片粘贴和拖拽不受影响
- 文档切换和导入导出不受影响
- 公众号复制链路可继续使用
